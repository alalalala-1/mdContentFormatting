# MD Content Formatting

Obsidian 插件，用于自动整理 Markdown 结构，提升长文档在 Reading / Live Preview 下的一致性与可读性。

## 项目目标

- 统一标题层级，减少手工维护成本
- 规范空行与段落边界，降低渲染异常概率
- 优化图片与列表在 Obsidian 中的显示体验

## 主要功能

- 标题层级规范化
  - 支持递进编号标题（如 `4.1`、`4.1.1`）自动映射层级
  - 非递进前缀标题（如 `1.`、`A.`、纯文本）按规则归类
  - 支持“小节后特例三级标题”语义识别（模糊匹配）
- 标题快修与正文 tag 识别
  - 支持生成/回写标题快修表
  - 支持将“中文标签 + 英文 tag”正文定义行从误判标题中恢复，并规范 `#标签` 形式
- 空行规范化
  - 标题、列表、表格、callout、分割线前后空行统一
  - 表格上方必须且只有一行空行，表格下方按普通块间距处理
  - 处理图片与 callout 紧邻场景
- 公式规范化
  - `$$` 起始行强制左对齐（去除前导空白）
- 无编号列表可读性优化
  - 嵌套层级符号与显示增强
  - Live Preview 模式下可视区分优化
- 图片显示优化
  - 插图居中
  - 自适应宽度（保留手动宽度优先）

## 使用方式

安装插件后，在 Obsidian 命令面板执行：

- `Format current note (content only)`
- `Generate/refresh heading review table`
- `Apply heading review table to headings`

说明：

- `Format current note (content only)`
  - 只整理正文内容，不改标题层级
- `Generate/refresh heading review table`
  - 生成或刷新标题快修表，方便集中审阅标题修正建议
- `Apply heading review table to headings`
  - 将标题快修表里的编辑结果回写到正文标题

插件会对当前笔记执行格式化与显示优化。

## 标题判定优先级

- 顶层章节锚点优先：`# 7`、`# 7.` 这类标题优先视为新的 H1 章节
- 显式编号路径其次：`## 4.1`、`### 4.1.2` 这类标题按编号深度映射层级，并在必要时做同级顺排或父路径补全
- 中文序号锚点再其次：如 `五、第三族` 且后文出现 `5.1 / 5.1.1` 家族时，会提升为对应顶层章节
- 语义特例只作用于章节内部：如“学习主线回顾”“容易混淆的点”“本节小结”“术语表”等，只在编号小节内部的普通标题上收敛为摘要小节，不会覆盖 H1 章节锚点
- 普通无编号标题最后处理：仅在缺少显式编号时，基于父章节路径做自动补号或层级收敛

## 标题实现结构

- `src/headingNormalizationShared.ts`
  - 放置标题编号解析、中文序号解析、标题前缀剥离、特殊语义识别等共享基础能力
- `src/headingNormalizationCore.ts`
  - 放置标题分类、章节状态对象、输出推导、状态更新等核心归一化逻辑
- `src/formatter.ts`
  - 负责编排整体格式化流程，并调用标题归一化核心模块

## 标题前缀家族

- 当前标题系统会先把前缀解析成统一 marker，再进入归一化状态机
- 当前支持的家族与典型形态如下：

| 家族 | 典型形态 | 当前角色 |
| --- | --- | --- |
| 中文序号 | `一`、`一、` | 章节锚点 / 中文家族入口 |
| 阿拉伯单段 | `1`、`1.` | 顶层章节锚点 |
| 阿拉伯路径 | `1.1`、`1.1.`、`1.1.1` | 结构化编号路径 |
| upper alpha 单段 | `A`、`A.`、`A)`、`A).` | 弱显式或有序小点 |
| lower alpha 单段 | `a`、`a.`、`a)`、`a).` | 弱显式或有序小点 |
| alpha 复合路径 | `A.1`、`A.1.1`、`a.2` | alpha 家族结构化路径 |
| 无编号标题 | 普通文本标题 | 依赖上下文做层级收敛或自动补号 |

## 标题语义强度

| 强度 | 含义 | 典型例子 |
| --- | --- | --- |
| `structured` | 可参与章节锚点、显式路径、父路径补全、自动补号 | `1`、`1.1`、`五、`、`A.1` |
| `ordered_explicit` | 明确是有序小点，但不直接等同于阿拉伯结构化路径 | `A.`、`A)`、`a.`、`a)` |
| `weak_explicit` | 只有显式前缀语义，主要用于避免误判 | `A`、`a` |
| `none` | 无前缀标题，按上下文做层级收敛或自动补号 | 普通文本标题 |

## 当前能力边界

| 范围 | 当前状态 | 说明 |
| --- | --- | --- |
| 中文锚点 / 阿拉伯路径 / upper-lower alpha 混排 | 已稳定支持 | 支持多家族混排、层级收敛与自动补号 |
| upper alpha → lower alpha 结构扩展 | 已稳定支持 | 例如 `A.` 下接 `a)` |
| alpha sibling 顺排 | 已稳定支持 | 如 `A. / C. / D)` 收敛为 `A. / B. / C)` |
| alpha 复合路径 | 已稳定支持 | 如 `A.1 / A.1.1 / A.1.1.1` |
| apply 后再次 format 稳定性 | 已稳定支持 | 标题快修表回写后不应重新打坏既有结构 |
| 多字母 alpha 段 | 暂未建模 | 如 `AA.1` |
| 更复杂的 alpha 复合重排 | 保守处理 | 如 `A.1 / B.2 / C.3` 的跨家族重排 |
| 更长的 `alpha + arabic + alpha` 复合路径 | 尚未完整建模 | 仍以当前局部规则为主 |
| 自由文本标题的深语义推断 | 保守处理 | 仍优先按上下文修复，不强推额外语义 |

## 标题规则矩阵

| 矩阵分组 | 当前覆盖重点 |
| --- | --- |
| 章节锚点 | 顶层单段数字章节保持 H1、带句点 H1 不误降级、纯文本 H1 保持顶层、章节内无编号标题自动补号 |
| 跨位数章节 | `9 → 10`、`30 → 31`、`99 → 100` 不误挂到前一章子节之下 |
| 语义特例与混排 | 摘要型标题收敛、中文序号与阿拉伯家族对齐、upper/lower alpha 混排、alpha 复合路径归一化 |
| 表格空行 | 表格上方固定一行空行，列表/callout/标题后接表格不双重插空；表格后接标题/列表/callout 保持普通间距 |
| 回写稳定性 | apply 标题快修表后再次 format，顶层章节、mixed family 与 alpha 复合路径保持稳定 |

## 标题维护约定

- 新增标题规则前，先判断它属于：
  - 章节锚点
  - 显式编号路径
  - 中文序号锚点
  - alpha 序列 / alpha 复合路径
  - 语义特例
  - 普通无编号标题
- 新增或修复标题逻辑时，优先先补矩阵测试，再改实现
- 如果新规则只影响章节内部摘要标题，不应覆盖已经明确写成 H1 的章节锚点
- 如果规则修改会影响 apply 后再 format 的行为，需要补一条回写稳定性测试
- 如果规则引入新的编号家族，优先先补 shared marker parser 测试，再补混排矩阵
